# Система миграций БД с HTTP API

## HTTP API Команды

### `GET /migrate/create`
Создаёт новую миграцию на основе изменений в моделях, но не применяет её к базе данных.

### `GET /migrate/apply`
Применяет последнюю созданную миграцию к базе данных.

### `GET /migrate/rollback`
Откатывает последнюю применённую миграцию, используя сгенерированный down_sql.

### `GET /migrate/status`
Показывает текущее состояние миграций и различия между схемой БД и моделями.

### `GET /migrate/log`
Выводит историю всех миграций с информацией о времени применения.
